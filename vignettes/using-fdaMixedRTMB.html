Using fdaMixedRTMB: boundary-value models, diagnostics, and marginal means

fdaMixedRTMB fits operator-based functional mixed models where each observed curve is represented on a common time grid. This vignette uses the packaged beer-tracker data to show the typical workflow:

  1. We turn event-level records into cumulative and weekly trajectory data.
  2. Fit a boundary-value model with fdaLm_rtmb().
  3. Inspect the fitted object with package methods.
  4. Use diagnostic plots.
  5. Use emmeans for the boundary-value regression.
  6. Extract fitted curves for plotting or downstream summaries.
library(fdaMixedRTMB)
library(emmeans)
#> Welcome to emmeans.
#> Caution: You lose important information if you filter this package's results.
#> See '? untidy'
library(ggplot2)

old_theme <- theme_set(theme_minimal(base_size = 11))
sex_cols <- c(F = "#D55E00", M = "#0072B2")

Prepare Beer-Tracker Data

The example data are event records: one row per logged beer. A functional response needs one row per subject and time point, so the first step is to aggregate to daily counts and fill missing subject-day combinations with zero.

Full-Period Curves

Full-period curves accumulate beer gains from the first analysis day through the last available day. The right endpoint of such a curve is the subject’s total beer gain over the full tracker period.

full_period_df <- transform(
  beer_daily,
  time = day,
  y = beer_gain,
  curve = name
)

full_curve_meta <- unique(full_period_df[c("curve", "subject", "name", "sex")])
full_curve_meta <- full_curve_meta[order(full_curve_meta$subject), ]

ggplot(full_period_df, aes(time, y, colour = sex)) +
  geom_line(aes(group = name), alpha = 0.3, linewidth = 0.4) +
  stat_summary(aes(group = sex), fun = mean, geom = "line", linewidth = 1.1) +
  scale_colour_manual(values = sex_cols) +
  labs(
    title = "Full-period cumulative beer-gain curves",
    x = "Day of tracker period",
    y = "Cumulative beers",
    colour = NULL
  )

A full-period model uses the same interface. For example, the endpoint boundary derivative could be modeled by sex:

fit_full <- fdaLm_rtmb(
  y | curve ~ 0,
  data = full_period_df,
  design = full_curve_meta,
  boundary_formula = ~ sex,
  boundary_value = TRUE,
  operator = operator_laplace(),
  right_boundary = right_boundary_operator_k1(alpha = c(0, 1)),
  time_variable = "time"
)

Fit a Boundary-Value Model

The model below uses monthly cumulative beer gains as the response. For a faster and more readable vignette, the fit uses four anonymized participants; the data-preparation code above created the full set of monthly curves.

model_subjects <- levels(subject_lookup$name)[1:4]

curve_meta_model <- subset(curve_meta, name %in% model_subjects)
curve_meta_model$curve <- factor(
  as.character(curve_meta_model$curve),
  levels = as.character(curve_meta_model$curve)
)

beer_monthly_model <- subset(beer_monthly_df, name %in% model_subjects)
beer_monthly_model$curve <- factor(
  as.character(beer_monthly_model$curve),
  levels = levels(curve_meta_model$curve)
)

c(
  observations = nrow(beer_monthly_model),
  curves = nlevels(beer_monthly_model$curve),
  days_per_curve = n_days_month
)
#>   observations         curves days_per_curve 
#>            560             20             28
fit <- fdaLm_rtmb(
  monthly_cumsum | curve ~ 0,
  data = beer_monthly_model,
  design = curve_meta_model,
  boundary_formula = ~ month_name,
  boundary_value = TRUE,
  operator = operator_laplace(),
  right_boundary = right_boundary_operator_k1(alpha = c(0, 1)),
  time_variable = "day_within_month"
)

The main arguments are:

  • monthly_cumsum | curve ~ 0: the response is monthly_cumsum, and curve identifies the functional curve. The fixed-effect part is 0 because this example puts the month effect in the boundary-value regression.
  • boundary_value = TRUE: estimate a nonzero right-boundary value.
  • boundary_formula = ~ month_name: model the curve-level boundary value as a function of month.
  • right_boundary = right_boundary_operator_k1(alpha = c(0, 1)): define the right boundary operator as endpoint derivative, so the fitted rho is the month-end derivative.
  • operator = operator_laplace(): use the K.order = 1 pure Laplace operator L f = -lambda_d2 f''. The shifted operator is available via operator_k1(lambda_start = c(1, 1)).

Plot Diagnostics

The S3 diagnostic method is available as plot(fit, which = c("fit", "residual", "rho")). The vignette below builds the same common diagnostics with ggplot2 so the plot style is consistent:

  • "fit": observed values against fitted values,
  • "residual": residuals against fitted values, and
  • "rho": estimated curve-specific right-boundary values.
diagnostic_df <- data.frame(
  observed = fit$data$y,
  fitted = as.numeric(predict(fit, type = "response")),
  residual = as.numeric(residuals(fit))
)

rho_df <- data.frame(
  curve = names(coef(fit, component = "rho")),
  rho = as.numeric(coef(fit, component = "rho"))
)
rho_df$curve <- factor(rho_df$curve, levels = rho_df$curve)

ggplot(diagnostic_df, aes(fitted, observed)) +
  geom_point(alpha = 0.65, size = 1.2) +
  geom_abline(slope = 1, intercept = 0, colour = "grey40") +
  labs(
    title = "Observed vs fitted",
    x = "Fitted value",
    y = "Observed value"
  )


ggplot(diagnostic_df, aes(fitted, residual)) +
  geom_hline(yintercept = 0, colour = "grey40") +
  geom_point(alpha = 0.65, size = 1.2) +
  labs(
    title = "Residuals vs fitted",
    x = "Fitted value",
    y = "Residual"
  )

Weekly Trajectories Without Cumulative Sums

The cumulative curves are useful when the scientific question is about total gain by the end of a period. A different question is whether the weekly numbers themselves change over time. In that case each curve is one person’s tracker-week beer totals, the time grid is week number, and the response is weekly_beers rather than an accumulated total. Weeks are consecutive 7-day blocks anchored at January 1. The observation window ends on May 28, which creates one trailing zero-only day; that day is folded into the final tracker week so the last plotted point is the May 21-28 week rather than an artificial zero-only week.

For non-cumulative weekly counts, the endpoint value is just the beer count in the final modeled week. The endpoint derivative is more useful: it estimates the local trend at the end of the observed period, in weekly beers per week. This example therefore uses right_boundary = c(0, 1) and a boundary regression rho ~ sex.

beer_weekly_daily <- beer_daily
raw_week_num <- ((beer_weekly_daily$day - 1L) %/% 7L) + 1L

last_raw_week <- max(raw_week_num)
last_raw_week_total <- sum(beer_weekly_daily$beer[raw_week_num == last_raw_week])
if (last_raw_week_total == 0 && last_raw_week > 1L) {
  raw_week_num[raw_week_num == last_raw_week] <- last_raw_week - 1L
}

beer_weekly_daily$week_num <- raw_week_num
n_weeks <- max(beer_weekly_daily$week_num)

beer_weekly_df <- aggregate(
  beer ~ subject + name + sex + week_num,
  beer_weekly_daily,
  sum
)
names(beer_weekly_df)[names(beer_weekly_df) == "beer"] <- "weekly_beers"
beer_weekly_df <- beer_weekly_df[order(beer_weekly_df$subject, beer_weekly_df$week_num), ]
beer_weekly_df$week_index <- beer_weekly_df$week_num - 1L
beer_weekly_df$week_label <- factor(
  paste("Week", beer_weekly_df$week_index),
  levels = paste("Week", seq_len(n_weeks) - 1L)
)
beer_weekly_df$curve <- beer_weekly_df$name

weekly_curve_meta <- unique(
  beer_weekly_df[c("curve", "subject", "name", "sex")]

Known Limitations

At the time of writing, user-specified grouped random effects are not part of the public modeling workflow. The practical supported pattern is to use the formula interface for ordinary fixed effects and to use boundary_formula for curve-level boundary-value regressions. The serial functional effect is handled internally through the operator-projection machinery.
